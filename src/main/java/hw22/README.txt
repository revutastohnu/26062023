Метод refuel() в класі GasStation синхронізовано (synchronized), щоб уникнути гонки потоків. Це означає, що в один момент часу лише один потік може виконувати цей метод.

Метод addToQueue() також синхронізований. Коли нова машина додається в чергу, всі потоки, які чекають (в стані wait), отримують сигнал про продовження роботи (notifyAll()).

Змінна fuelTank визначена як volatile. Це означає, що зміни, зроблені одним потоком, будуть негайно видимі всім іншим потокам.

У методі refuel(), якщо черга пуста або немає достатньо пального, потік переходить в стан очікування (wait()). Якщо в чергу додається нова машина або з'являється пальне, очікуючі потоки будуть відновлені (notifyAll()).
